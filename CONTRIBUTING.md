# Contributing

Thanks for contributing.

## Ground Rules

- Never commit secrets (OAuth client JSON, token files).
- Do not add logging that prints message bodies or tokens.
- Keep changes minimal and well-tested.

## Development

```bash
npm install
npm run test
npm run build
```

## Pull Requests

- Include a clear description of the behavior change and why.
- Add or update tests when changing parsing, safety checks, or tool behavior.
